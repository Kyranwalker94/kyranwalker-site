Armijos Calendar 🗓️

An invite-only dinner booking calendar for Armijos, publicly viewable availability, invite-code gating, and a booking request flow built entirely in vanilla JS with no backend required.

The idea

Coordinating dinner plans with family and friends across group chats is messy, people asking when we're free, double-booking, forgotten replies. I wanted a simple, single place where availability is always visible and the booking process takes care of itself.

The calendar lives on the Armijos site and is designed to feel part of the brand rather than a bolted-on tool. Anyone can view it. Only people with an invite code can request a booking.

How it works

Available dates are set in the calendar configuration, currently all Thursdays and Saturdays across May and June 2026. Each date is shown as one of four states:

  • Available, open and bookable with an invite code
  • Event, a pre-scheduled event (e.g. Family Dinner) where invites are sent directly
  • Taken, no longer available
  • Unavailable, not a configured date (default for most days)

Clicking an available date opens a booking modal. The guest enters their name and invite code. Valid codes are pre-configured per person or family group, each one maps to a display name (e.g. "WALKER01" → "Walker Family"). If the code is correct, the booking is saved to localStorage and a pre-formatted confirmation message is generated for the guest to send to the host.

The tech

Fully static, HTML, CSS, and vanilla JS, no framework, no backend. Availability data is defined in a single JS config object at the top of the page, making it easy to update dates and add invite codes without touching any other code. Bookings are stored in the browser's localStorage, which gives the guest a persistent "Yours ✓" state on their device.

  • Month navigation between configured months
  • Past dates automatically greyed out using JS date comparison
  • Today marker (red dot) on current date
  • Admin panel at ?admin URL param, shows all localStorage bookings in a table
  • Enter key support and full keyboard-accessible modal
  • Responsive layout down to small mobile screens

What's next

The current version is intentionally lightweight, it coordinates bookings via a copied message rather than automating them, which suits the personal scale of the project. Future additions being considered:

  • Email or WhatsApp notification on booking submission
  • Multi-guest events with RSVP tracking per invite code
  • Host dashboard with approval/decline controls
  • Optional backend to sync bookings across devices in real time
